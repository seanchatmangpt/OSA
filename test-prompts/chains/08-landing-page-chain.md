# Chain 08: SaaS Landing Page (4 messages)

Tests: pure design quality, CSS skills, responsive, visual polish, no complex logic

**KEY: This chain is ALL about design quality. Does it look like a real product or a template?**

---

## Message 1 — Hero + Nav
```
Landing page for "Flowboard" - a team collaboration tool. Navbar and hero section. Hero should have a headline, subtitle, two CTA buttons, and a product screenshot mockup. Make it look premium.
```

**First Principles Check:**
- Did it create a visually impressive hero (not just centered text)?
- Did it add a gradient or visual treatment to the background?
- Did the navbar have proper items (logo, links, CTA button)?
- Did the CTA buttons have different visual weight (primary + secondary)?
- Did it create a mock product screenshot (CSS-drawn dashboard or placeholder)?
- Did it choose a good color palette (not default blue)?
- Did it import a proper font (Inter, Plus Jakarta, etc.)?
- Did the overall look feel premium or generic?
- Did it set up proper spacing system (consistent padding/margins)?

---

## Message 2 — Features + Social proof
```
Add features section with 6 cards and a social proof bar showing company logos that trust us.
```

**First Principles Check:**
- Did it create 6 feature cards with icons, titles, and descriptions?
- Did the feature descriptions sound like real marketing copy?
- Did the cards have hover effects?
- Are the cards in a responsive grid (3x2 → 2x3 → 1x6)?
- Did it add social proof below with company names?
- Did it style the company logos (text-only is fine, but styled)?
- Did it add a "Trusted by X teams" heading?
- Is there enough whitespace between sections?
- Did each section have a heading + subtitle?

---

## Message 3 — Pricing + FAQ
```
Pricing section with 3 tiers: Free, Pro, Enterprise. Pro should be marked as popular. Then add an FAQ accordion below.
```

**First Principles Check:**
- Did it create 3 pricing cards with features lists?
- Did the "Popular" card stand out (elevated, border, badge)?
- Did pricing look realistic (not $1, $2, $3)?
- Did each tier have a clear CTA button?
- Did the FAQ accordion actually open/close?
- Did it animate the accordion (smooth expand/collapse)?
- Did it show +/- or arrow icons on FAQ items?
- Did FAQ questions sound realistic?
- Only one FAQ open at a time? (common pattern)

---

## Message 4 — CTA + Footer + Full responsive
```
Final CTA section with email signup, footer with 4 columns of links and social icons. Make everything work on mobile.
```

**First Principles Check:**
- Did the CTA section have a distinct background (gradient, color)?
- Did the email input + button sit side by side?
- Did the footer have organized link columns?
- Did it add social media icons?
- Did it add a copyright line at the bottom?

RESPONSIVE (the real test):
- Does navbar become hamburger on mobile?
- Does hamburger menu actually open/close?
- Does hero text resize properly on mobile?
- Do feature cards stack properly?
- Do pricing cards stack vertically on mobile?
- Does footer go from 4 columns → 2x2 → stacked?
- Does the CTA email input stack below button on mobile?
- Is text still readable at all sizes?
- Are tap targets big enough on mobile (44px min)?

OVERALL DESIGN QUALITY:
- Would this page look out of place next to Stripe/Linear/Vercel?
- Is there a consistent visual language throughout?
- Does it use spacing, typography, and color consistently?
- Are transitions/hovers smooth and purposeful?
