# OSA — Optimal System Agent

> Multi-agent orchestration runtime in Elixir/OTP. Signal Theory-optimized.
> Local-first, open source, BEAM-powered.

## Build & Run

```bash
mix setup                              # deps.get + ecto.setup + compile
mix test                               # full suite
mix test test/tools/                    # tool tests only
mix test test/providers/                # provider tests only
mix test --no-start                    # unit tests without app boot
mix test --include integration          # integration tests
mix compile --warnings-as-errors        # zero-tolerance compilation
mix osa.serve                          # start HTTP API server
```

**Default port:** 8089 (override with `OSA_HTTP_PORT`)

## Architecture

```
lib/optimal_system_agent/
  agent/          ReAct loop, context builder, hooks, guardrails
  channels/       HTTP (Plug/Bandit), WebSocket, CLI, Telegram, Discord, Slack
  events/         Bus (Goldrush BEAM bytecode), PubSub, DLQ, circuit breakers
  providers/      Ollama, Anthropic, OpenAI-compat, OpenRouter, MIOSA, custom
  tools/          25 built-in tools + registry with JSON Schema validation
  memory/         SQLite + ETS + episodic, SICA pattern engine, skills
  swarm/          parallel, pipeline, debate, review_loop coordinators
  signal/         Signal classifier S=(M,G,T,F,W), noise filter
  consensus/      HotStuff BFT consensus protocol
  mcp/            MCP client (GenServer) + server (stdio/HTTP)
  healing/        Reflex arcs, process fingerprinting, org evolution
  commerce/       Agent marketplace
  supervisors/    Infrastructure, Sessions, AgentServices, Extensions
```

**Supervision:** `:rest_for_one` top-level, 4 subsystem supervisors.
**In-memory state:** ETS tables (30+), created in `Application.start/2`.
**Databases:** SQLite (local) + PostgreSQL (platform multi-tenant).
**HTTP stack:** Bandit 1.6 + Plug (no Phoenix framework dependency).
**Event routing:** Goldrush compiled BEAM bytecode dispatch.

## Code Standards

- Elixir 1.17+ / OTP 27+
- `mix format` — no warnings in compilation
- OTP patterns: GenServer, DynamicSupervisor, Task.Supervisor, Registry
- ETS for hot-path state; SQLite for persistence
- Jason for JSON (use plain maps `%{}` for JSON-compatible keys, not keyword lists)
- Tool registration via `Tools.Registry` with `ex_json_schema` validation
- Signal Theory encoding S=(Mode, Genre, Type, Format, Weight) on all agent I/O
- Config in `~/.osa/.env` (generated by setup wizard)
- Skills are hot-reloadable markdown files in `~/.osa/skills/`
