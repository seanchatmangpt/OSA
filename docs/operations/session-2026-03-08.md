## Timeline

### 06:07 — `24d1b30` refactor: remove Go TUI, harden auth + sandbox, clean up codebase

**What was done:**
- Removed `priv/go/tui-v2/` — the old Go TUI (Bubble Tea) that was fully replaced by the Rust TUI. ~23,000 lines deleted.
- Wired SSE 401 auto-refresh: when the SSE stream gets a 401, the app now attempts token refresh via `ApiClient.try_refresh_token()` and reconnects automatically. Previously it just showed a toast telling the user to `/login` manually.
- Added `self.sidebar.set_provider_info()` at both `ModelSwitched` and `OnboardingComplete` handlers in `handle_backend.rs`. The header and status bar were updating on model switch but the sidebar was missed.
- Fixed 3 dead clauses: removed 2 unreachable `{:error, :not_found}` patterns in `platform_routes.ex`, wrapped `Task.async` in try/rescue in `auto_fixer.ex` so the `{:error, reason}` clause is reachable.
- Deleted 26 stale `.beam` files from the project root that were causing "redefining module" compilation warnings.
- Gated `.env` file loading in `config/runtime.exs` on `config_env() != :test` — the `.env` file was being loaded during `mix test`, overriding `http_port: 0` with `OSA_HTTP_PORT=4001` and injecting Supabase DATABASE_URL, causing port conflicts and PG circuit breaker errors.
- Consolidated duplicate `System.get_env("DATABASE_URL")` reads in `runtime.exs`.
- Added non-root `osa` user to the Dockerfile runner stage (addgroup/adduser, chown, USER directive).
- Deleted duplicate `.dockerignore 2` file.
- Sorted models by `context_window` descending in `data_routes.ex`.
- Updated README: accurate line counts (112K not 200K), codebase breakdown table, supervision tree reflecting 4 subsystem supervisors, test badge (1958), Docker non-root mention, SSE auto-refresh mention.

**What was NOT done (deferred):**
- SSE auto-refresh inside the SSE client itself (we used app-level handling instead — cleaner, less invasive).
- Full test coverage for Sandbox subsystem (Executor, Config, Pool, Provisioner modules have 0 tests — ~60% coverage).
- Fixing the 8 pre-existing `TripleStore` compilation warnings (from `miosa_knowledge` dep, not our code).
